# GPU 显存模型

## 推理中的显存分配

```
┌─────────────────────────────┐
│  模型权重（静态）            │  启动时加载
│  70B FP16 = 140 GB          │
├─────────────────────────────┤
│  KV Cache（动态）            │  随请求增长
│  ~20-40 GB                  │
├─────────────────────────────┤
│  激活显存（动态）            │  ~2-5 GB
├─────────────────────────────┤
│  预留 / 碎片                 │  ~5-10%
└─────────────────────────────┘
```

## OOM 常见原因

1. Batch size 太大 → 降低 max_num_seqs
2. 序列太长 → 降低 max_model_len
3. 显存碎片 → 用 vLLM（PagedAttention）
