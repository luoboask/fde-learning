# Agent 系统与架构

> FDE 需要支持 Agent 场景：记忆、工具调用、多 Agent 协作

## Agent 基础架构

### 什么是 Agent

Agent = 大模型 + 记忆 + 工具 + 规划能力

```
┌─────────────────────────────────┐
│         Agent Core              │
│  ┌──────────┐  ┌──────────────┐ │
│  │ Planning  │  │  Memory      │ │
│  │ (规划)    │  │  (记忆系统)   │ │
│  └────┬─────┘  └──────┬───────┘ │
│       │               │         │
│  ┌────┴─────┐  ┌──────┴───────┐ │
│  │ Tool Use  │  │  Knowledge   │ │
│  │ (工具调用) │  │  (知识库/RAG) │ │
│  └──────────┘  └──────────────┘ │
└─────────────────────────────────┘
```

## 记忆系统设计

### 记忆分层

```
短期记忆（Session Memory）
  ├─ 当前对话上下文
  ├─ 临时变量和状态
  └─ 生命周期：单次会话

长期记忆（Persistent Memory）
  ├─ 用户画像和偏好
  ├─ 历史对话摘要
  ├─ 知识库索引
  └─ 生命周期：持久化存储

工作记忆（Working Memory）
  ├─ 当前任务上下文
  ├─ 中间推理结果
  └─ 生命周期：单个任务
```

### 向量数据库选型

| 数据库 | 部署方式 | 规模 | 特点 |
|--------|---------|------|------|
| Milvus | 自建 | 亿级 | 生产级、支持分布式 |
| Qdrant | 自建/云 | 千万级 | Rust 实现、高性能 |
| Pinecone | SaaS | 亿级 | 全托管、零运维 |
| Chroma | 本地/嵌入 | 百万级 | 轻量、开发友好 |
| Weaviate | 自建/云 | 千万级 | GraphQL 接口 |

### RAG 部署架构

```
用户请求 → Embedding 服务 → 向量检索 → Top-K 结果
                                       ↓
用户请求 + 检索结果 → LLM 推理服务 → 最终回答
```

## 工具调用与 Function Calling 部署

### Function Calling 原理

```
用户: "北京明天天气怎么样？"
  ↓
LLM 输出: {
  "name": "get_weather",
  "arguments": {"city": "北京", "date": "明天"}
}
  ↓
执行工具: 调用天气 API
  ↓
工具结果: {"temp": 25, "condition": "晴"}
  ↓
LLM 生成最终回答
```

### 部署要点

- Function Calling 需要 LLM 原生支持（OpenAI API 格式）
- 工具调用的延迟叠加（每个工具调用增加网络延迟）
- 多步工具调用的超时和重试

## Agent 可观测性

### 关键监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| Agent 调用链 | 完整执行步骤追踪 | 单任务超过 30 步 |
| 工具调用成功率 | 工具调用的成功/失败比 | 低于 95% |
| Token 消耗 | 每次任务的 Token 用量 | 单次超过 100K |
| 响应延迟 | 端到端延迟 | 超过 30s |

## 面试视角

### 常考问题

1. **"你了解 Agent 的架构吗？"**
   - Agent = LLM + Memory + Tools + Planning
   - 记忆分层：短期/长期/工作记忆
   - 工具调用机制：Function Calling / ReAct

2. **"记忆系统怎么设计？"**
   - 向量数据库存储长期记忆
   - RAG 检索增强生成
   - 记忆压缩和摘要策略

3. **"多 Agent 协作怎么实现？"**
   - 角色分配和职责划分
   - 通信协议（消息传递）
   - 任务分解和协调
