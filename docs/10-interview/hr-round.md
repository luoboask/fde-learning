# HR 面策略

> 通过 HR 面的核心是展现稳定性、文化匹配和职业规划清晰，同时合理争取薪资

---

## HR 面的四大考察重点

### 1. 稳定性

HR 最关心的是：你会不会在入职后短期内离职？

**评估信号**：
- 过往工作经历：每段工作是否稳定（频繁跳槽是红灯）
- 离职原因是否合理（被裁员 vs 主动寻求发展）
- 对新公司的了解程度（是否真的感兴趣）
- 家庭/生活因素（是否可能影响稳定性）

**应对策略**：
- 离职原因聚焦"追求更大发展空间"，避免负面评价前公司
- 展现你对新公司/新岗位的深入了解（做过功课）
- 表达长期深耕的意愿

### 2. 文化匹配

HR 会评估你是否和公司的价值观、工作风格匹配。

**评估信号**：
- 沟通风格是否自然真诚
- 对工作节奏和压力的态度
- 团队协作意识
- 是否认同公司的使命和方向

**应对策略**：
- 提前了解公司文化和价值观（看官网、员工分享、脉脉等）
- 在回答中体现和公司价值观的一致性
- 保持真诚，不要刻意迎合

### 3. 薪资预期

HR 需要在公司预算和你的预期之间找到平衡点。

**评估信号**：
- 你当前薪资水平
- 你对薪资的期望和弹性
- 你对非薪资福利的关注度

**应对策略**：
- 提前调研市场行情（看 Boss 直聘、脉脉等）
- 给出一个合理区间而非固定数字
- 关注总包（base + bonus + RSU + 福利），不只看 base

### 4. 职业规划

HR 希望你的职业规划和岗位发展路径是匹配的。

**评估信号**：
- 你对未来 3-5 年的规划
- 你期望的成长方向（技术深耕 vs 管理发展）
- 你选择机会的标准

**应对策略**：
- 展现清晰的职业规划（"我希望在 FDE 方向深耕，3 年内成为技术专家"）
- 说明为什么这个岗位能帮你实现规划
- 体现你有长期思考，不是随便看机会

## HR 面常见问题及回答策略

### Q1：离职原因

**标准回答框架**：
```
"在 [公司] 的 [X] 年收获很大，在 [方向] 积累了完整的经验。
现在我希望在一个更聚焦 AI 部署的赛道承担更大的责任，
而贵公司在 [方向] 的布局和我想要深耕的方向非常匹配。"
```

**禁忌**：
- ❌ "晋升太难"（暗示你能力不够）
- ❌ "和领导关系不好"（暗示你不好合作）
- ❌ "待久了想换"（暗示你不稳定）
- ❌ "薪资太低"（虽然可能是事实，但不是最好的表达）

### Q2：你为什么选择我们公司？

**标准回答框架**：
```
"三个原因：
第一，贵公司在 [AI 部署/推理方向] 的投入力度很大，这正是我想深耕的方向。
第二，我了解到团队在 [具体技术点] 上有深入实践，我很希望参与其中。
第三，这个岗位既能发挥我的技术深度，也能用到我积累的管理经验。"
```

**注意事项**：
- 提前做公司调研（官网、技术博客、员工分享）
- 不要用万能模板，要体现你做了功课
- 如果知道面试官的名字/背景，可以提及（"我看到 XX 的技术分享..."）

### Q3：你当前薪资是多少？期望多少？

**标准回答框架**：
```
"我当前的总包是 [X]（base + bonus + 其他），期望涨幅在 [20-30]%。
不过我更关注的是整体发展机会，薪资是其中一部分，
岗位成长空间和团队氛围对我来说同样重要。"
```

**注意事项**：
- 给出区间而非固定数字
- 表达弹性，不要显得僵化
- 强调"总包"概念，不只是 base salary

### Q4：你最大的优势是什么？

**标准回答框架**：
```
"我的优势在于三个交叉：
- 技术深度和工程能力的交叉：既理解底层原理，也有生产环境经验
- 技术和管理能力的交叉：既能做技术决策，也能带团队推进
- 经验和学习能力的交叉：有扎实基础，也保持对前沿技术的持续跟进

这三个交叉让我在 FDE 岗位上能快速产生价值，并且有长期成长空间。"
```

### Q5：你最大的不足是什么？

**标准回答框架**：
```
"如果说不足的话，我在 [某个非核心技能] 上的实践还比较少。
比如 [Triton kernel 编程 / 特定框架]，我理解原理但还没有亲手写过。
不过我已经在 [具体行动：看课程、做实验]，预计 [时间] 内可以补齐。"
```

**注意事项**：
- 选一个**非核心能力**的不足（不要说"我不懂量化"）
- 同时给出**改进计划**
- 体现你有**自我认知**和**持续学习**的习惯

### Q6：你手头有其他 offer 吗？

**标准回答框架**：
```
"我目前在看几个机会，包括 [方向 A] 和 [方向 B] 的岗位。
不过贵公司是我优先级最高的选择，因为 [具体理由]。
如果这边能给 offer，我会认真考虑并尽快给回复。"
```

**注意事项**：
- 不要说"只有你们一家"（显得你竞争力不足）
- 也不要说"有好几个 offer 在等"（可能让 HR 觉得你不诚意）
- 表达贵公司是你的优先选择

### Q7：你未来 3-5 年的职业规划是什么？

**标准回答框架**：
```
"我希望在 FDE 方向深耕，3 年内成长为推理部署领域的技术专家。
具体来说：
- 技术上：持续跟进推理引擎和量化的前沿，成为团队的技术骨干
- 管理上：继续提升团队管理能力，带更大的团队
- 业务上：更深入地理解业务需求，让技术服务于业务目标

贵公司的这个岗位和我这个规划高度匹配，所以我非常希望加入。"
```

### Q8：你能接受加班吗？

**标准回答框架**：
```
"我在之前的工作中也经历过冲刺阶段，理解在关键节点加班是必要的。
我相信只要是为了项目交付，适度加班没问题。
同时我也认为长期来看，通过流程优化和技术手段提升效率，
比单纯靠加班更可持续。"
```

**注意事项**：
- 不要直接拒绝（"不接受加班"）
- 也不要无底线承诺（"完全没问题，加班是我的常态"）
- 体现你有**弹性**但也关注**效率和可持续性**

## 薪资谈判技巧

### 谈判前的准备

1. **了解市场行情**：通过 Boss 直聘、脉脉、内推朋友等了解同类岗位的薪资范围
2. **明确自己的底线**：设定一个最低可接受数字
3. **了解公司薪酬结构**：base、bonus、RSU、签字费、各项福利

### 谈判策略

```
第一轮（HR 询问预期）：
"我当前的总包是 [X]，期望涨幅在 20-30% 之间。不过我更关注整体发展机会。"

第二轮（HR 给数字）：
如果数字低于预期："感谢您的 offer。这个数字和我的预期有一定差距，
考虑到我在 [具体方向] 的经验和能为团队带来的价值，是否还有调整空间？"

第三轮（拉锯）：
如果 base 无法调整，关注其他维度："理解 base 有标准，
那 bonus、RSU、签字费、年假等其他方面是否可以再讨论？"
```

### 谈判注意事项

- ✅ 用数据说话（当前薪资、市场水平、你的价值）
- ✅ 保持礼貌和专业，不要让谈判变成对抗
- ✅ 关注总包，不要只盯着 base
- ❌ 不要虚报当前薪资（可能被 background check 发现）
- ❌ 不要用其他 offer 作为威胁（除非是真的）
- ❌ 不要因为薪资差距一点就轻易拒绝，考虑整体发展

## 反问环节怎么提问

### 好的反问示例

反问环节是展示你对岗位真实兴趣的好机会，以下是几个高质量问题：

**技术方向**：
- "团队目前主要用哪个推理引擎？自研还是开源方案？"
- "这个岗位前三个月最重要的 deliverable 是什么？"
- "团队目前最大的技术挑战是什么？"

**团队方向**：
- "团队的规模和结构是怎样的？"
- "团队的技术分享和知识传承机制是怎样的？"

**业务方向**：
- "这个岗位服务的业务线有哪些？优先级如何排序？"
- "公司对 AI 部署方向的未来 1-2 年规划是什么？"

**个人成长**：
- "这个岗位的成长路径是怎样的？"
- "公司对技术人员的培养机制有哪些？"

### 不要问的问题

- ❌ "薪资福利怎么样？"（这是 HR 面之前就该谈好的，反问环节问这个显得格局小）
- ❌ "加班多吗？"（虽然可以理解，但不宜在反问环节问）
- ❌ "面试结果什么时候出？"（可以问 HR，但不适合问技术面试官）
- ❌ 不问任何问题（会显得你对岗位不感兴趣）

### 反问环节的时间分配

| 问题类型 | 建议数量 | 目的 |
|----------|----------|------|
| 技术方向 | 1-2 个 | 展示技术热情和深度思考 |
| 团队/业务 | 1 个 | 了解工作环境和业务前景 |
| 个人成长 | 1 个 | 展现长期规划意识 |

## 面试视角：HR 面的评分表

面试官在 HR 面通常会填写以下评估表：

| 维度 | 权重 | 评估标准 |
|------|------|----------|
| 稳定性 | 25% | 离职原因合理、过往经历稳定、对新公司有了解 |
| 文化匹配 | 25% | 沟通真诚自然、认同公司价值观、协作意识强 |
| 职业规划 | 20% | 规划清晰、与岗位匹配、有长期思考 |
| 薪资合理 | 15% | 期望在市场范围内、有弹性 |
| 动机真实 | 15% | 对新岗位有真实兴趣，不是骑驴找马 |
